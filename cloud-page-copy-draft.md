# Cloud 页面文案稿（v2）

这版结合你给出的核心信息，重点突出两件事：

- 把组合后的工具发布为一套云端能力
- 通过 Cloud 统一管理安全、鉴权、权限和网关，并由 `oo-cli` 作为统一使用入口

---

## 页面要传达的核心

Cloud 不是“多一个入口”，而是“多一层统一管理”。

你在 Studio 里把工具组合好之后，发布到 Cloud：  
它会变成一套可持续运行、可统一管理的云端能力。  
发布后，无论你使用哪种 Agent，只要接入 `oo-cli`，就能继续调用这套工具。

## 中段模块：Cloud 在解决什么

### 模块标题

工具跑通后，Cloud 帮你把它变成可持续使用的一套能力

### 模块说明

本地跑通只是开始。  
要让工具长期稳定地被使用，你还要处理上线后的管理问题。  
Cloud 把这些统一承接，让你专注在工具本身。

### 卡片一：组合式能力云端化

**标题：** 组合后的工具，直接发布成一套云端能力  
**描述：** 你把已有工具组合完成后，可以直接发布到 Cloud。发布后是一整套可复用能力，不需要再为每个工具单独补一套系统。  
**要点：**

- 组合完成后可直接发布
- 一套能力持续复用

### 卡片二：跨 Agent 统一管理

**标题：** 安全、鉴权、权限、网关统一由 Cloud 承接  
**描述：** 无论你使用哪种 Agent，只要接入 `oo-cli`，背后都走同一套管理体系。账号和工具能力跟着你走，不绑定在单一 Agent 上。  
**要点：**

- 管理能力随个人账号走
- 换 Agent 也能沿用同一套工具

## 流程模块文案

### 步骤一（Cloud）

**标题：** 发布到 Cloud，把组合能力稳定跑在云端  
**描述：** 本地验证完成后，把工具发布到 Cloud。Cloud 统一承接上线后的管理层，你不需要分散维护。  
**短点：**

- 云端稳定运行
- 统一鉴权与权限
- 网关统一管理

### 步骤二（oo-cli）

**标题：** 发布后，通过 oo-cli 在不同 Agent 中继续使用  
**描述：** 使用者不需要关心部署细节，也不用改变习惯。接入 `oo-cli` 后，就能继续调用同一套工具能力。
