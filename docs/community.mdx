# Community

The community catalog is where you browse packages [published online by everyone](/docs/get-started/export-and-publish#online-publishing).

<Image
  sources={{
    light: useBaseUrl("/img/docs/community/community-page.png"),
    dark: useBaseUrl("/img/docs/community/community-page.png"),
  }}
  width="720"
/>

After clicking on any package, you can view its detailed information.

The page includes flow previews, package docs, descriptions, version information, and related publishing details.

Depending on how the package was published, the open button may behave differently.

<Image
  sources={{
    light: useBaseUrl("/img/docs/community/community-detail.png"),
    dark: useBaseUrl("/img/docs/community/community-detail.png"),
  }}
  width="720"
/>

Clicking the button opens the project locally. If it was [published as a template](/docs/get-started/export-and-publish#publish-as-template), it includes the full project and is ready for further editing. Otherwise, the opened project contains only the flows.

## GitHub Publisher

In actual engineering development, a more common way to update and publish is to publish automatically after code updates.

OOMOL Studio supports GitHub Actions, so you can publish automatically when code changes.

<Image
  sources={{
    light: useBaseUrl("/img/docs/community/community-publisher.png"),
    dark: useBaseUrl("/img/docs/community/community-publisher.png"),
  }}
  width="720"
/>

<Image
  sources={{
    light: useBaseUrl("/img/docs/community/community-publisher-config.png"),
    dark: useBaseUrl("/img/docs/community/community-publisher-config.png"),
  }}
  width="720"
/>

The GitHub repository field should contain your repository name. After you configure the publisher account, published packages will show that user as the publisher.

You also need to configure the corresponding workflow in your GitHub repository. You can use the [example project](https://github.com/oomol-lab/oopm-publish-action) as a reference.

## Collaboration

:::info
This feature is only available to paid subscribers. You can enter [OOMOL Console](https://console.oomol.com/) through the button at the bottom left of the application homepage, then change your subscription plan in the Console.

<Image
  sources={{
    light: useBaseUrl("/img/docs/community/community-console-entry.png"),
    dark: useBaseUrl("/img/docs/community/community-console-entry.png"),
  }}
  width="720"
/>
:::

By default, if you have already published a package, only you can publish new versions of that package.

If you need to allow other users to have publishing permissions, you can invite other users to become maintainers.

<Image
  sources={{
    light: useBaseUrl("/img/docs/community/community-maintainer.png"),
    dark: useBaseUrl("/img/docs/community/community-maintainer.png"),
  }}
  width="720"
/>

After clicking the `Invite` button, you can search for users and generate an invitation link. Send the invitation link to the specified user to let them accept becoming a maintainer.

<Image
  sources={{
    light: useBaseUrl("/img/docs/community/community-maintainer-invite.png"),
    dark: useBaseUrl("/img/docs/community/community-maintainer-invite.png"),
  }}
  width="720"
/>

:::info
All maintainers have the same permissions. Please make sure you invite the correct users.
:::
