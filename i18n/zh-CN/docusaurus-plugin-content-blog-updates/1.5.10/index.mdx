# OOMOL Studio

## 通用

### 1. OOPilot 更懂当前工作上下文

本次版本继续优化 OOPilot 在真实项目中的使用体验。它现在会更稳定地理解当前打开的文件、连续对话中的上下文，以及用户前后多轮请求之间的关系。

对于日常使用来说，这意味着当你让 OOPilot 修改 Block、补充配置、继续刚才的任务，或基于当前文件做调整时，它更容易抓住你正在处理的重点，减少重复解释需求的次数。

<!-- truncate -->

### 2. 优化 AI 助手的任务理解与提示质量

我们精简并调整了 OOPilot 内部提示内容，减少重复提示和无关上下文，让 AI 在处理工作流、Block、Package 和 Skill 相关任务时更聚焦。

在创建任务 Block、编写工作流配置、处理云端 Skill 文档和发布准备时，OOPilot 会给出更贴近 OOMOL 场景的建议，也会减少模板占位内容、自依赖配置、泛化说明等常见问题。

### 3. 修复新环境下的聊天启动状态

之前在部分新配置或新环境中，聊天功能可能已经可用，但界面仍然显示需要完成 setup。现在 OOMOL Studio 会在检测到可用的聊天能力后同步标记完成状态，避免右下角继续误显示 `Finish Setup`。

## 工作流与连接器

### 1. 连接器操作更清晰

本次版本完善了连接器相关的交互能力。工作流节点可以提供更直接的连接器操作入口，连接状态、菜单项和错误反馈也更明确。

当 Block 需要连接外部服务或账号时，用户可以更容易看到当前连接状态，并从节点附近完成相关操作，减少在不同页面之间来回查找配置入口的成本。

### 2. 改进连接器状态与错误提示

我们优化了 Designer 中连接器状态展示和节点错误处理。节点出现连接问题或执行前置条件不完整时，界面会更清楚地提示问题位置，帮助用户更快判断是连接未完成、配置缺失，还是服务状态异常。

### 3. 修复连接器菜单与样式问题

本次版本修复了 connector actions 和 connector submenu 的样式问题，让连接器菜单在不同节点和面板中展示更稳定，减少布局错位或交互不一致的情况。

## 编程功能

### 1. 更新 VS Code 上游能力

OOMOL Studio 已同步上游 VS Code `1.117.0`，带来编辑器底层能力、兼容性和依赖层面的更新。

### 2. 优化包搜索与包管理体验

包搜索现在会在有搜索关键词时按相关性排序，帮助用户更快找到目标 Package；没有关键词时则避免错误套用相关性排序，减少搜索结果不符合预期的情况。

我们也修复了 Package 下载过程中可能显示错误图标的问题，让安装和下载状态更准确。

### 3. 改进日志面板

日志面板调整为更接近 VS Code 的展示风格，并增加 reset 操作。调试工作流、查看执行日志、排查报错时，界面状态更容易恢复，也更适合长时间查看。

### 4. 修复聊天标题生成稳定性

本次版本调整了聊天标题生成时的输出限制，减少标题生成过程中出现异常输出或过长内容的风险。

## 其他

### 依赖更新

本次版本同步升级了多项 OOMOL 内部核心组件：

- OOPilot 升级到 `0.1.139`
- Studio Core 升级到 `0.0.899`
- Designer 升级到 `0.8.426`
- Oocana 升级到 `0.24.11`

这些更新共同提升了 AI 助手、工作流设计器、连接器服务和运行时能力的一致性。
