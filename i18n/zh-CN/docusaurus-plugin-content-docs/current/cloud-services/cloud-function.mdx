# 云函数

如果你想通过代码直接调用云函数，请继续看 **API 与 SDK** 下面的文档。对外的产品名称是“云函数”；但当前直连接入时，执行接口和 SDK 包名仍然沿用 `Cloud Task v3` 这个名字。

OOMOL 云函数是一层托管运行服务。你可以把已经在 OOMOL 里跑通的能力继续发布为线上 API，而不必自己再搭一套服务器、部署流程和运维体系。

## 概述

云函数的定位很直接：本地负责把能力做出来，云端负责把能力稳定跑起来。你负责实现逻辑，OOMOL 负责部署、扩缩容和运行维护。

### 核心优势

- **接入简单**：不需要额外搭建服务框架，就可以把能力发布到线上
- **按量付费**：只为实际用量付费，不需要预付服务器成本
- **运维负担低**：不需要自己维护服务器、环境和扩容策略
- **自动扩缩容**：流量上来时由平台自动承接

## 三种交付方式

同一份能力可以面向不同场景交付。下面三种方式不是同一层级的“部署模式”，而是三种对外提供能力的方式。

### 1. 发布为包

把你的区块发布成包，分享给别人使用，或者在更复杂的工作流里继续复用。

**使用场景：**
- 与其他开发者分享可复用组件
- 构建其他人可以扩展的模块化功能
- 为社区提供可继续组合的基础能力

**工作原理：**
- 打包你的区块，定义清晰的输入和输出
- 发布到 OOMOL 社区包库
- 其他人可以发现并在他们的工作流中使用你的区块

### 2. 部署为云函数

把已经验证过的能力直接交付成线上服务。

**使用场景：**
- 为应用提供可调用的 API
- 构建后端能力而不自己维护服务器
- 让已经跑通的函数进入真实线上使用

**功能特性：**
- 自动 HTTPS 端点
- 内置负载均衡
- 零停机部署
- 按需自动扩展

部署完成后，OOMOL 会提供实际的调用入口和集成信息。当前 API 形态和 SDK 命名，请参考 [云函数 API 概览](/zh-CN/docs/cloud-services/cloud-task/overview)。

### 3. 作为 MCP 能力提供给 AI 调用

通过模型上下文协议（MCP）发布你的函数，供 AI 助手或 Agent 调用。

**使用场景：**
- 让 AI 助手直接调用你的函数
- 提供 AI 可以发现和使用的工具
- 把你的能力接进 AI 驱动的工具链和工作流

**工作原理：**
- 你的函数通过 MCP 协议暴露
- AI 侧可以发现并调用你的函数
- 与支持 MCP 的 AI 助手无缝集成

## 定价

云函数采用按使用量付费的模式：

- **无固定费用**：没有月度服务器费用或预付款
- **按用量计费**：只为实际调用付费
- **价格可查**：计费规则可以直接在控制台查看
- **包含试用额度**：适合先验证再决定是否放量

访问 [OOMOL 控制台](https://console.oomol.com/cloud-function) 查看详细定价信息。

## 快速开始

### 前置条件

- 一个 OOMOL 账户
- 在计算机上安装 OOMOL Studio
- 准备好要部署的工作流

### 部署步骤

1. **创建工作流**
   - 在 OOMOL Studio 中构建工作流
   - 在本地跑通并确认结果
   - 定义清晰的输入和输出参数

2. **配置部署**
   - 在 OOMOL Studio 中打开部署面板
   - 选择“云函数”作为部署目标
   - 配置函数设置（超时、内存等）

3. **部署**
   - 点击“部署到云端”
   - 等待部署完成
   - 获取函数的调用信息

4. **测试函数**
   - 使用提供的调用入口或 SDK 测试函数
   - 在 OOMOL 控制台监控使用情况和性能
   - 再把 API 提供给用户或其他系统调用

### 部署示例

云函数当前通过 `Cloud Task v3` 这一接入层提供直连能力。若要查看可直接运行的请求示例，请参考 [HTTP + curl](/zh-CN/docs/cloud-services/cloud-task/curl-api) 文档。

## 最佳实践

### 函数设计

1. **保持函数专注**：每个函数尽量只负责一件事
2. **定义清晰的接口**：记录输入和输出参数
3. **优雅处理错误**：返回有意义的错误消息
4. **优化冷启动**：最小化初始化时间

### 安全

1. **使用 API 密钥**：通过身份验证保护函数
2. **验证输入**：始终验证和清理用户输入
3. **设置速率限制**：通过适当的速率限制防止滥用
4. **监控使用**：关注异常模式或潜在攻击

### 性能

1. **优化依赖**：只包含必要的库
2. **适时缓存**：减少冗余计算
3. **使用适当的超时**：设置现实的执行时间限制
4. **按目标负载测试**：在预期流量下验证性能

## 与传统部署的对比

| 功能 | 传统服务器 | OOMOL 云函数 |
|------|-----------|-------------|
| **启动时间** | 数小时到数天 | 几分钟 |
| **服务器管理** | 需要 | 无需 |
| **扩展** | 手动 | 自动 |
| **成本** | 固定月费 | 按使用付费 |
| **维护** | 持续需要 | OOMOL 负责 |
| **SSL/HTTPS** | 手动设置 | 内置 |
| **部署** | 复杂 | 一键完成 |
