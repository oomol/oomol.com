# 📝 面板说明

## 1. 整体布局

<img src={panel} width="760" />

## 2. 🟡 顶部面板

该面板主要用于运行工作流。

| 模块               | 描述                 |
| ------------------ | -------------------- |
| 运行流             | 运行当前工作流程     |
| 重新运行节点       | 运行选定的节点       |
| 在文本编辑器中打开 | 看工作流的 yaml 源码 |

## 3. 🔴 左面板

该面板主要用于创建配置工作流和块。

| 模块       | 描述                                                                                                 |
| ---------- | ---------------------------------------------------------------------------------------------------- |
| Flows      | 工作流画布，一个项目可以创建多个。同一个项目中的所有流依赖项都可以共享。                             |
| Blocks     | 产品的核心模块，包含 Scriptlet Block 和 Shared Block 两种。 Shared Block 由 Scriptlet Block 转化而来 |
| 工作区     | 源代码。                                                                                             |
| 源码搜索   | 源代码全文搜索                                                                                       |
| git        | 源代码管理工具                                                                                       |
| 编辑器插件 | 支持 vscode vsix 格式插件                                                                            |
| 存储       | 存储工作流中需要使用的静态资源的位置。静态资源包括图像、视频和表格内容。                             |
| 密钥       | 工作流中的某些块需要调用第三方服务。可以使用 Secrets 模块访问调用服务所需的机密。                    |

## 4. 🔵 底部面板

该面板主要用于工程师的深度开发和调试工作流程。

| 模块           | 描述                                                                                                                                                 |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| 终端           | 连接到容器 OVM 的终端。需要注意的是，在 OVM 中运行的系统是 Linux。                                                                                   |
| 代码依赖管理   | 用于安装 Python 和 JS 的第三方库的面板。需要注意的是，通过面板安装的 Python 库会自动添加到 `requirements.txt` 中。直接使用终端安装的用户需要手动添加 |
| 工作流日志     | 工作流运行期间生成的日志。                                                                                                                           |
| 工具包安装日志 | 该工作流依赖从 oomol 社区下载的工具包，此处会自动安装                                                                                                |
| 初始化日志     | 在 OVM 容器中首次启动项目时需要自动执行的脚本。                                                                                                      |

| 问题 | 编码错误 |
| 端口 | 用于外部访问 OVM 里面的服务 |

## 5. ⚪ 右面板

该面板主要用于参考现成的积木。

面板上方为封装，下方为封装中的功能模块，需要注意的是，如果没有选择，则会列出所有的封装和功能模块。

| 模块                           | 描述                                                             |
| ------------------------------ | ---------------------------------------------------------------- |
| 已经安装的 Packages            | OOMOL 使用过程中安装或者预装的工具包，在此使用                   |
| Package 商店                   | 您可以在 Packages Store 中下载其他用户分享的工具包               |
| 发布                           | 发布工具包                                                       |
| AI 聊天助手 OOMOL Copilot Chat | 遇到开发使用问题可以问 OOMOL Copilot, OOMOL 会以对话的方式帮助你 |
| AI 编程助手 OOMOL Copilot Edit | OOMOL 开发助手，可以帮助编写以及优化代码                         |
