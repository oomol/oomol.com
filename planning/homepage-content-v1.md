## 2. 能力范围区块

### 区块标题

不只是接现成工具，也能封装你自己的能力

### 区块说明

OOMOL 既能连接常见 SaaS 和已有 API，也能把你自己的接口、脚本、内部系统和业务逻辑组合成 Agent 可调用的能力。

### 卡片 1

#### 标题

Connect SaaS

#### 描述

连接常见 SaaS 服务，把已有工具能力接入到你的使用流程中。

### 卡片 2

#### 标题

Bring APIs

#### 描述

接入带 auth 或 API Key 的服务，让外部接口也能成为 Agent 可调用的能力。

### 卡片 3

#### 标题

Build Custom Logic

#### 描述

组合你自己的 API、脚本、内部系统和业务逻辑，不被现成工具范围限制。

## 3. 路径区块

### 区块标题

从使用到构建，一条连续路径

### 区块说明

你可以先直接使用已有能力；当现成能力不够，再进入 Studio 自己生成、组合和发布。

### 路径 A

#### 标题

先使用已有能力

#### 描述

通过 oo-cli 在 Codex、Claude Code 或终端中搜索、查看并运行已发布能力，快速开始，不必先搭底层集成。

#### 按钮

Start with oo-cli

### 路径 B

#### 标题

再构建自己的能力

#### 描述

在 OOMOL Studio 中生成和组合能力，在本地验证后，通过 Cloud 运行并交付给自己、团队或客户使用。

#### 按钮

Build with Studio

## 4. 使用入口区块

### 区块标题

同一份能力，多种使用入口

### 区块说明

同一个能力，不必重复实现。可以先给开发者在 CLI 中使用，也可以接成 API、封装成 MCP，或者继续承载到图形界面中。

### 入口 1

#### 标题

CLI

#### 描述

让 Codex、Claude Code 和终端环境直接调用能力，适合最快开始和日常工作流使用。

### 入口 2

#### 标题

API

#### 描述

把同一份能力接入到你自己的应用、服务或自动化代码中。

### 入口 3

#### 标题

MCP

#### 描述

把能力作为 MCP 工具接入外部 Agent framework 和 AI 工具生态。

### 入口 4

#### 标题

App Surface

#### 描述

把能力承载到图形界面中，让非终端用户也能直接使用。

## 5. 开发者价值区块

### 区块标题

开发者不该为了交付能力而切换一整套工作方式

### 区块说明

OOMOL 不是让你适应一套新的平台工作方式，而是把能力生成、能力组合、本地验证和交付重新带回开发者熟悉的工作流里。

### 要点 1

#### 标题

真实编码环境

#### 描述

不是在受限的表单或黑盒配置里拼装逻辑，而是在真实编码环境中完成开发和验证。

### 要点 2

#### 标题

不用先学新的 DSL

#### 描述

你关注的是能力本身，而不是先学习另一套平台语言。

### 要点 3

#### 标题

可以组合，也可以扩展

#### 描述

既能使用现成能力，也能继续封装、扩展和组合你自己的实现。

### 要点 4

#### 标题

一份实现，多处复用

#### 描述

同一个能力可以服务 CLI、API、MCP 和更多入口，而不必重复搭建多套实现。

### 要点 5

#### 标题

支持 skill 模式扩展

#### 描述

能力不只是一次性调用，还可以继续沉淀成可复用、可扩展的 skill 资产。

## 7. Cloud 区块

### 区块标题

当能力需要持续运行和交付时，Cloud 来承接

### 区块说明

把运行时、配置、Secrets、订阅和使用数据统一放到 Cloud 中管理。需要时，能力在云端运行，用户只使用入口，不必接触底层实现。

### 要点 1

#### 标题

统一运行时

#### 描述

让能力从本地验证走向持续运行时，不需要再围着同一份实现额外搭一层后端。

### 要点 2

#### 标题

配置与 Secrets 管理

#### 描述

把配置、权限和 Secrets 集中管理，减少分散维护成本。

### 要点 3

#### 标题

交付与订阅

#### 描述

把能力交付给自己、团队或客户，并在同一个后台里承接运行和使用关系。

### 要点 4

#### 标题

数据与可观察性

#### 描述

统一查看运行配置和使用数据，让交付后的管理可持续。

### 要点 5

#### 标题

代码保持在你的控制范围内

#### 描述

当能力在云端承接运行时，用户消费的是入口，而不是直接拿到底层实现。

### 按钮

Explore Cloud
